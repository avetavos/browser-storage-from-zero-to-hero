## Object Store คืออะไร?

**Object store** คือสิ่งที่ IndexedDB ใช้แทนตารางในฐานข้อมูล แต่ละเรคคอร์ดในอ็อบเจกต์สโตร์คือ JavaScript object ที่คุณเก็บเข้าไป — แต่ต่างจากตาราง SQL ตรงที่ไม่มีการบังคับโครงสร้าง คุณเก็บออบเจกต์ที่มีรูปแบบต่างกันในสโตร์เดียวกันได้ กฎโครงสร้างเพียงอย่างเดียวคือวิธีกำหนด **key**

เปรียบได้ดังนี้:

- แถวในตาราง SQL มีชุดคอลัมน์ที่ตายตัว
- เรคคอร์ดใน object store คือ JS object ใดก็ได้ที่คุณส่งเข้าไป บวกกับ key ที่ระบุตัวตนของมัน

## การสร้าง Object Store

คุณสร้าง object store โดยเรียก `db.createObjectStore(name, options)` บน instance ของฐานข้อมูล เมธอดนี้ **ต้อง** เรียกใช้ภายใน event handler `onupgradeneeded` เท่านั้น — ใช้ได้เฉพาะในช่วงการอัปเกรดเวอร์ชัน

```js
const request = indexedDB.open('my-db', 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  db.createObjectStore('contacts', { keyPath: 'id' });
};
```

อาร์กิวเมนต์แรกคือชื่อสโตร์ อาร์กิวเมนต์ที่สองคือออบเจกต์ตัวเลือกที่ควบคุมวิธีการทำงานของ key

## keyPath: In-Line Key

เมื่อกำหนด `keyPath` สโตร์จะใช้ **property บนออบเจกต์ที่เก็บ** เป็น key ซึ่งเรียกว่า **in-line key** เพราะ key อยู่ภายในออบเจกต์นั้นเอง

```js
// ออบเจกต์ทุกชิ้นที่เก็บ ต้องมี property `id`
db.createObjectStore('contacts', { keyPath: 'id' });

// เรคคอร์ดที่ถูกต้อง:
store.add({ id: 1, name: 'Alice', email: 'alice@example.com' });

// ไม่ถูกต้อง — ไม่มีฟิลด์ `id` จะเกิด error:
store.add({ name: 'Bob' });
```

หากกำหนด `autoIncrement: true` ร่วมกับ `keyPath` เบราว์เซอร์จะสร้าง key ให้อัตโนมัติและเขียนลงในออบเจกต์ที่ตำแหน่งที่ระบุ:

```js
db.createObjectStore('contacts', { keyPath: 'id', autoIncrement: true });

// `id` จะถูกกรอกอัตโนมัติ:
store.add({ name: 'Alice' }); // เก็บเป็น { id: 1, name: 'Alice' }
```

## Out-of-Line Key

เมื่อไม่กำหนด `keyPath` (หรือตั้งเป็น `null`) key จะ **แยกออกจากออบเจกต์** เรียกว่า **out-of-line key** คุณต้องส่ง key เองเป็นอาร์กิวเมนต์ที่สองของ `add()` หรือ `put()`

```js
db.createObjectStore('raw-blobs', { keyPath: null });

// คุณต้องระบุ key เอง:
store.add({ data: '...' }, 'blob-001');
```

ผสมกับ `autoIncrement: true` เพื่อให้เบราว์เซอร์สร้าง key เป็นจำนวนเต็มโดยไม่แตะออบเจกต์ที่เก็บ:

```js
db.createObjectStore('logs', { autoIncrement: true });

// ไม่ต้องใส่ key — เบราว์เซอร์กำหนดเป็น 1, 2, 3, ...
store.add({ level: 'info', message: 'App started' });
```

## ตารางเปรียบเทียบโหมด Key

| โหมด | keyPath | autoIncrement | แหล่งที่มาของ Key |
|---|---|---|---|
| In-line (กำหนดเอง) | `'id'` | `false` | ฟิลด์ `id` ของออบเจกต์ |
| In-line (อัตโนมัติ) | `'id'` | `true` | สร้างอัตโนมัติ เก็บไว้ใน `id` |
| Out-of-line (กำหนดเอง) | (ไม่มี) | `false` | ผู้เรียกส่ง key มาเอง |
| Out-of-line (อัตโนมัติ) | (ไม่มี) | `true` | สร้างอัตโนมัติ ไม่อยู่ในออบเจกต์ |

## ลองทำ: สร้าง Object Store สองอัน

ตัวอย่างด้านล่างเปิดฐานข้อมูลที่เวอร์ชัน 1 และสร้างสโตร์สองอัน:

- `contacts` — in-line key โดยใช้ property `id`
- `logs` — out-of-line key แบบ auto-increment

รันดูเพื่อดูชื่อสโตร์ทั้งสองถูกพิมพ์ออกมาใน console

export const objectStoresCode = `
const db = await new Promise((resolve, reject) => {
  const req = indexedDB.open('demo-stores', 1);
  req.onupgradeneeded = (e) => {
    const d = e.target.result;
    d.createObjectStore('contacts', { keyPath: 'id' });
    d.createObjectStore('logs', { autoIncrement: true });
    console.log('Stores created');
  };
  req.onsuccess = (e) => resolve(e.target.result);
  req.onerror = (e) => reject(e.target.error);
});
console.log('Object stores:', Array.from(db.objectStoreNames).join(', '));
db.close();
indexedDB.deleteDatabase('demo-stores');
console.log('Cleanup done');
`.trim();
