## สองวิธีในการเขียนไฟล์

เบราว์เซอร์มีพื้นที่เขียนไฟล์สองแบบ:

1. **File System Access API** (`showOpenFilePicker`, `showSaveFilePicker`) — ให้ผู้ใช้เลือกไฟล์จริงจาก file system ในเครื่อง ต้องใช้ user gesture (คลิกปุ่ม) ไม่รองรับในทุกเบราว์เซอร์
2. **Origin Private File System (OPFS)** — virtual file system แบบ sandboxed scoped ตาม origin ไม่ต้องใช้ user gesture, ไม่มี file picker, เขียน program ได้ทั้งหมด ทำงานใน worker ใช้ quota เหมือน IndexedDB

## File System Access API

ฟังก์ชัน picker จะแสดง dialog ไฟล์ของ OS ต้องเรียกจาก user-gesture handler

```js
// Open a file chosen by the user
async function openFile() {
  // showOpenFilePicker must be called inside a user gesture (click, etc.)
  const [fileHandle] = await window.showOpenFilePicker();
  const file = await fileHandle.getFile();
  const text = await file.text();
  console.log('File contents:', text.slice(0, 200));
}

// Save / overwrite a file
async function saveFile(content) {
  const handle = await window.showSaveFilePicker({
    suggestedName: 'output.txt',
    types: [{ description: 'Text file', accept: { 'text/plain': ['.txt'] } }],
  });
  const writable = await handle.createWritable();
  await writable.write(content);
  await writable.close();
}
```

เนื่องจาก `showOpenFilePicker` และ `showSaveFilePicker` ต้องใช้ user gesture จึงไม่สามารถรันใน StorageRunner ข้างต้นได้โดยอัตโนมัติ ตัวอย่าง OPFS ด้านล่างรันได้เต็มรูปแบบ

## Origin Private File System (OPFS)

OPFS ให้ directory tree ส่วนตัวแก่ทุก origin ไม่ต้องใช้ user gesture — แค่เรียก `navigator.storage.getDirectory()`

```js
// Get the root of the OPFS
const root = await navigator.storage.getDirectory();

// Create or open a file handle
const fileHandle = await root.getFileHandle('notes.txt', { create: true });

// Write to it
const writable = await fileHandle.createWritable();
await writable.write('Hello from OPFS!');
await writable.close();

// Read it back
const file = await fileHandle.getFile();
const text = await file.text();
console.log(text); // 'Hello from OPFS!'
```

## การนำทางใน directory

```js
const root = await navigator.storage.getDirectory();

// Create a sub-directory
const subDir = await root.getDirectoryHandle('drafts', { create: true });

// List entries
for await (const [name, handle] of subDir.entries()) {
  console.log(handle.kind, name);
}
```

## รันได้: OPFS write และ read

export const opfsCode = `// OPFS is fully available in the same-origin iframe — no user gesture needed.
const root = await navigator.storage.getDirectory();

// Write a file
const fh = await root.getFileHandle('demo-opfs.txt', { create: true });
const writable = await fh.createWritable();
await writable.write('Hello from OPFS! Written at: ' + new Date().toISOString());
await writable.close();
console.log('File written.');

// Read it back
const file = await fh.getFile();
const text = await file.text();
console.log('Read back:', text);

// List all files in the root
console.log('Root entries:');
for await (const [name, handle] of root.entries()) {
  console.log(' ', handle.kind, name);
}

// Clean up — remove the demo file
await root.removeEntry('demo-opfs.txt');
console.log('Cleaned up demo-opfs.txt');`;

<Callout title="ประเด็นสำคัญ">
พื้นที่จัดเก็บ OPFS นับรวมกับ quota รวมของ origin (pool เดียวกับ IndexedDB และ Cache API) ใช้ `navigator.storage.estimate()` เพื่อตรวจสอบพื้นที่ที่มี ข้อมูล OPFS จะอยู่รอดหลังรีสตาร์ทเบราว์เซอร์ แต่สามารถถูก evict โดยเบราว์เซอร์ภายใต้แรงกดดันของพื้นที่จัดเก็บ หาก origin ไม่ได้รับ persistent storage ผ่าน `navigator.storage.persist()`
</Callout>
