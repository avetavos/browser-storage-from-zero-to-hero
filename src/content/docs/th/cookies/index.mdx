## cookie คืออะไร?

cookie คือ string ขนาดเล็ก — แต่ละชิ้นมีขนาดไม่เกินประมาณ 4 KB — ที่ browser จัดเก็บแยกตาม domain และ**ส่งไปพร้อมกับ HTTP request ทุกครั้งโดยอัตโนมัติ** นี่คือสิ่งที่ทำให้ cookie แตกต่างจากกลไกจัดเก็บข้อมูลอื่นๆ ใน browser ทั้งหมด

เมื่อ server ส่ง `Set-Cookie` response header มา browser จะเก็บ cookie นั้นและส่งกลับไปใน request ถัดๆ ไปที่ตรงกับ domain นั้น (ขึ้นอยู่กับ attribute ของ cookie เช่น `Path`, `Domain`, `Secure`, และ `SameSite`)

เมื่อ JavaScript เรียก `document.cookie = 'name=value'` มันจะตั้งค่า cookie ฝั่ง client — แต่ browser จะยังคงส่ง cookie นั้นไปยัง server ใน request ถัดไปอีกด้วย

## cookie ที่ตั้งฝั่ง client vs ฝั่ง server

มีสองวิธีที่ cookie จะถูกเก็บไว้ใน cookie jar ของ browser:

**ตั้งโดย server (ผ่าน `Set-Cookie` header):**

```http
HTTP/1.1 200 OK
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict
```

Server เป็นผู้ควบคุมค่าและ attribute ของ cookie cookie ที่มี `HttpOnly` จะมองไม่เห็นจาก JavaScript เลย — `document.cookie` จะไม่แสดงมันออกมาไม่ว่ากรณีใด

**ตั้งโดย client (ผ่าน `document.cookie`):**

```js
document.cookie = 'theme=dark; Max-Age=86400; SameSite=Lax';
```

JavaScript สามารถตั้งค่า อ่าน และลบ cookie ที่ไม่ได้ทำเครื่องหมาย `HttpOnly` ได้

## cookie vs Web Storage: เลือกใช้เมื่อไหร่

คำถามสำคัญที่สุดคือ: **server จำเป็นต้องรับค่านี้หรือเปล่า?**

| กรณีการใช้งาน | ตัวเลือกที่ดีที่สุด | เหตุผล |
|---|---|---|
| Auth token / session ID | Cookie (HttpOnly + Secure) | Server ต้องการมัน; HttpOnly ป้องกันการขโมยผ่าน XSS |
| CSRF token | Cookie | Server ตั้งค่า; JS อ่านเพื่อใส่ใน request body |
| ค่าความชอบของผู้ใช้ (ธีม, ภาษา) | Web Storage | ใช้ฝั่ง client เท่านั้น; ไม่จำเป็นต้องส่งไป server ทุก request |
| ตะกร้าสินค้า (client-only) | Web Storage หรือ IndexedDB | ข้อมูลมากกว่า รูปแบบสมบูรณ์กว่า; ไม่มีภาระทางเครือข่าย |
| Feature flags จาก server | Cookie | Server ปรับ response ได้ก่อน JS โหลด |

กฎพื้นฐาน: **ใช้ cookie สำหรับทุกสิ่งที่ server ต้องรับ; ใช้ Web Storage สำหรับ UI state ที่เป็น client-only**

<Callout title="cookie vs Web Storage">
เลือกใช้ cookie เมื่อ **server** ต้องการค่านั้น — authentication token, session ID, ค่าความชอบที่ server อ่าน ส่วน `localStorage` หรือ `sessionStorage` เหมาะสำหรับค่าที่เป็น **client-only** — UI state, cached JSON, feature toggle ที่ไม่จำเป็นต้องส่งถึง server การส่งข้อมูลที่ไม่จำเป็นไปกับทุก request สิ้นเปลืองแบนด์วิธและอาจเปิดเผยข้อมูลสำคัญได้
</Callout>
