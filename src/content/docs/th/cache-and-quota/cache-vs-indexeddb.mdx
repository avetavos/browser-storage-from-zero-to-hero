## เลือก storage mechanism ที่เหมาะสม

เบราว์เซอร์มี storage API หลักสามตัวสำหรับข้อมูลถาวร แต่ละตัวมีวัตถุประสงค์ต่างกันและเสริมกัน — สิ่งสำคัญคือต้องเลือกให้ถูกกรณี

| Storage | เหมาะสำหรับ | ประเภทข้อมูล | Async? | ขนาดโดยประมาณ |
|---|---|---|---|---|
| **Cache API** | HTTP response, static asset | Request/Response | ใช่ | ขนาดใหญ่ (GB ได้) |
| **IndexedDB** | ข้อมูลแอปที่มีโครงสร้าง, ข้อมูลออฟไลน์ | JS object, blob | ใช่ | ขนาดใหญ่ (GB ได้) |
| **localStorage** | สตริง config ขนาดเล็ก, preferences | สตริงเท่านั้น | ไม่ (synchronous) | ~5 MB |

## Cache API

Cache API จัดเก็บคู่ `Request`/`Response` มีรูปแบบเหมือน HTTP — key คือ URL และ value คือ Response เหมาะสำหรับ:

- Pre-caching static asset (HTML, CSS, JS, รูปภาพ) เพื่อใช้ออฟไลน์
- Runtime caching ของ API response ภายใน Service Worker
- ให้บริการเนื้อหา stale ขณะดึงข้อมูลใหม่ในพื้นหลัง

Cache API มีประโยชน์สูงสุดเมื่อใช้ร่วมกับ Service Worker ดูรูปแบบการแคชด้วย Service Worker ได้ที่ [PWA course](https://avetavos.github.io/pwa-from-zero-to-hero/en/service-workers/)

## IndexedDB

IndexedDB จัดเก็บ JavaScript value ใด ๆ (object, array, blob, typed array) ใน object store พร้อม index ที่เลือกได้ เหมาะสำหรับ:

- ข้อมูลแอปออฟไลน์: ข้อมูลผู้ใช้, ข้อความ, เอกสาร
- Blob ขนาดใหญ่: เสียง, วิดีโอ, รูปภาพที่เป็นส่วนหนึ่งของโมเดลข้อมูลแอป
- ข้อมูลที่ต้องการ query, เรียงลำดับ หรือกรองฝั่ง client

## localStorage

`localStorage` จัดเก็บคู่ key/value ที่เป็นสตริงแบบ synchronous โดยบล็อก main thread ทุกครั้งที่อ่านหรือเขียน เหมาะสำหรับ:

- Preference ขนาดเล็ก: เลือก theme, ภาษา, token เดียว
- ค่าที่อ่านทุกครั้งที่โหลดหน้า โดยยอมรับการเข้าถึงแบบ synchronous
- ข้อมูลที่มีขนาดไม่กี่ kilobyte

ห้ามจัดเก็บออบเจกต์ขนาดใหญ่หรือเขียนบ่อยใน `localStorage` เนื่องจากจะทำให้ rendering หยุดชะงัก

<Callout title="จุดสำคัญ">
ใช้ Cache API สำหรับทุกสิ่งที่มีรูปแบบเหมือน HTTP response (URL + headers + body) ใช้ IndexedDB สำหรับ record ของแอปพลิเคชันและ blob ที่ต้องการ query หรืออัปเดต ใช้ localStorage เฉพาะสำหรับ string preference ขนาดเล็กที่ต้องอ่านแบบ synchronous เมื่อเริ่มต้น สำหรับกลยุทธ์ offline ใน PWA มักใช้ Cache API และ IndexedDB ร่วมกัน — asset ใน Cache API และข้อมูลแอปใน IndexedDB
</Callout>
